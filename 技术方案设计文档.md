## 1. 方案概述

### 1.1 目标

基于《申请企业基本情况报告-V2.xlsx》，构建一套线上数据填报系统，涵盖 14 个核心业务模块，替代传统 Excel 手工填报方式，实现企业基本信息、人员情况、管理制度、设备资源、研发业绩等多维度数据的数字化采集与管理。

### 1.2 选型原则

| 原则 | 说明 |
|------|------|
| **免费 & 开源** | 所有核心技术栈均采用 MIT / Apache 2.0 / BSD 等开源协议，无商业授权费用 |
| **成熟 & 稳定** | 选用社区活跃、有长期维护历史、生产验证充分的框架和工具 |
| **便捷** | 开发体验友好，中文生态完善，降低上手与维护成本 |
| **轻量化** | 避免过度工程化，最小化依赖，可在低配服务器（2C4G）上流畅运行 |

## 6. 模块功能设计

### 6.1 用户认证模块

- **注册/登录**：用户名 + 密码登录，密码 bcrypt 哈希存储
- **Token 机制**：JWT Access Token（30分钟）+ Refresh Token（7天）
- **角色**：`enterprise`（企业填报用户）、`admin`（管理员）
- **会话安全**：Token 存储于 `httpOnly Cookie` 或 `localStorage`（前端可配置），防 CSRF

### 6.2 填报工作台（Dashboard）

- 展示当前企业填报进度（14个模块完成情况）
- 未完成草稿提醒（登录后弹窗询问是否恢复）
- 已提交报告列表（状态标签：草稿 / 待提交 / 已提交 / 待修改 / 已驳回）
- 快捷入口：继续填报、查看历史报告

### 6.3 数据填报模块

#### 表单控件规范

| 数据类型 | 控件 | 验证规则 |
|----------|------|----------|
| 字符串 | `el-input` | 非空验证、maxlength、格式正则 |
| 数值 | `el-input-number` | 非负数值、范围、小数位数 |
| 日期 | `el-date-picker` (type=date) | 日期格式、有效期逻辑 |
| 日期区间 | `el-date-picker` (type=daterange) | 开始日期 ≤ 结束日期 |
| 长文本 | `el-input` (type=textarea) | maxlength=2000 |
| 百分比 | `el-input-number` + 后缀"%" | 0-100 范围 |
| 单选 | `el-radio-group` | 必选验证 |
| 多选 | `el-checkbox-group` / `el-select` (multiple) | 至少选一项 |
| 自动计算 | `el-input` (readonly) | 基于其他字段自动计算，只读显示 |
| 文件上传 | `el-upload` | 格式/大小校验（见文件上传模块）|

#### 交互规范

- 字段标签与 Excel 文档保持一致，提供 tooltip 说明和示例
- 实时验证（`blur` 事件触发），错误信息显示在字段下方
- 条件联动字段（如：选择"是"后显示附加字段，使用 `v-show` 实现）
- 分步填报：每个模块独立页面，顶部展示步骤进度条（`el-steps`）

### 6.4 草稿管理模块

#### 自动保存

```javascript
// composables/useAutoSave.js
// 每 3 分钟自动 POST /api/v1/drafts/{report_id}/{module_code}
// 使用 useIntervalFn (VueUse) 定时触发
// 页面离开时（beforeunload / Vue Router beforeRouteLeave）弹出确认对话框
```

#### 草稿恢复

- 登录后检测未完成草稿 → 弹窗提示 → 一键恢复（含已上传文件）
- "草稿管理"页面：按保存时间倒序展示历史草稿列表

### 6.5 文件上传模块

#### 前端上传组件（基于 `el-upload`）

```
文件校验流程：
① 前端校验文件格式（MIME 类型白名单）
② 前端校验文件大小（文档类 ≤20MB，图片 ≤10MB）
③ 上传到后端 /api/v1/files/upload
④ 后端再次验证（防绕过前端校验）
⑤ 存入 MinIO，记录到 attachments 表
⑥ 返回文件 ID，前端保存至表单数据
```

#### 文件存储规范

- **存储路径**：`{bucket}/{company_credit_code}/{year}/{module_code}/{stored_name}`
- **文件命名**：`{企业信用代码}_{YYYYMMDD}_{原文件名}`
- **访问控制**：MinIO 存储桶设为私有，通过后端签发 **预签名 URL**（有效期 1 小时）供前端访问/预览

#### 支持格式

| 类型 | 格式 | 大小限制 |
|------|------|----------|
| 文档类 | `.pdf`, `.doc`, `.docx` | ≤ 20MB |
| 表格类 | `.xls`, `.xlsx` | ≤ 20MB |
| 图片类 | `.jpg`, `.jpeg`, `.png`, `.gif` | ≤ 10MB |

### 6.6 报告提交模块

#### 提交流程

```
① 点击"提交" → 前端全量数据校验（必填项、格式）
② 校验通过 → POST /api/v1/reports/{id}/submit
③ 后端再次校验 → 生成报告编号（格式：CR-{年份}-{公司缩写}-{序号}）
④ 更新报告状态为"已提交"
⑤ 写入操作日志
⑥ 前端展示"提交成功"页面，提供"下载 PDF""打印"入口
```

#### 重复提交

- 支持基于历史报告"复制新建"，减少重复录入
- 每次提交生成独立报告编号和版本号
- 可配置提交间隔限制（默认 30 天，管理员可调整）

#### 状态流转

```
草稿 → 待提交 → 已提交 ─→ 待修改（管理员要求修改）→ 已提交
                    └→ 已驳回（管理员驳回）
```

### 6.7 数据修改模块

- **草稿状态**：所有字段完全开放修改，实时保存至草稿
- **已提交状态**：企业用户点击"申请修改"→ 填写修改原因（可选）→ 重新进入填报状态 → 重新提交后版本号递增
- **修改记录**：自动记录字段级变更（字段名、修改前值、修改后值、操作人、时间），不可删除

### 6.8 管理员模块

- 查看所有企业的报告提交状态
- 按企业名称、提交状态、时间范围筛选
- 驳回报告 / 要求修改（附说明）
- 查看操作审计日志
- 导出报告数据（CSV/Excel）
- 用户管理（创建/禁用账号，分配角色）

## 8. 安全设计

### 8.1 认证与授权

- **JWT 认证**：Access Token 有效期 30 分钟，Refresh Token 有效期 7 天
- **RBAC 权限**：FastAPI Depends 实现角色验证，企业用户只能访问本企业数据
- **密码策略**：最小长度 8 位，需包含字母和数字，bcrypt 哈希存储（cost factor = 12）
- **登录限制**：连续失败 5 次锁定账号 15 分钟（slowapi + Redis 或内存计数）

### 8.2 数据安全

- **传输加密**：全站 HTTPS（Nginx 配置 TLS 1.2+）
- **SQL 注入防护**：全程使用 SQLAlchemy ORM 参数化查询，禁止拼接 SQL
- **XSS 防护**：
  - 后端响应设置 `Content-Security-Policy` 头
  - 前端 Vue 模板自动转义，避免使用 `v-html`
  - 输入长度限制、特殊字符过滤
- **CSRF 防护**：JWT Bearer Token 方式天然防 CSRF（不依赖 Cookie 自动携带）
- **文件安全**：
  - 后端校验文件 Magic Bytes（非仅靠扩展名）
  - 文件通过 MinIO 预签名 URL 访问，不暴露真实路径
  - 拒绝可执行文件格式（.exe, .sh, .py 等）

### 8.3 审计日志

记录以下关键操作，保留至少 6 个月：

| 操作类型 | 记录内容 |
|----------|----------|
| 登录/登出 | 用户 ID、时间、IP、结果 |
| 创建/提交报告 | 用户、报告 ID、时间 |
| 数据修改 | 用户、字段、修改前后值、时间 |
| 文件上传/删除 | 用户、文件名、报告 ID、时间 |
| 管理员操作 | 操作类型、目标、时间 |

### 8.4 接口限流（slowapi）

```python
# 登录接口：每 IP 每分钟最多 10 次
# 上传接口：每用户每分钟最多 20 次
# 普通接口：每用户每分钟最多 200 次
```

minio:
